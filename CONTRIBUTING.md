# Contributing to Curated AI Voice Tools

Thank you for your interest in contributing! This project aims to maintain a high-quality, curated list of AI voice and audio tools.

## How to Contribute

### Suggesting a Tool

1. **Check if the tool already exists** - Search the README to make sure the tool is not already listed.
2. **Open an issue** - Use the [Tool Suggestion](https://github.com/awesome-ai-tools/curated-ai-voice-tools/issues/new?template=tool-suggestion.yml) template.
3. **Or submit a PR** - Follow the guidelines below.

### Submitting a Pull Request

1. Fork the repository.
2. Create a new branch: `git checkout -b add-tool-name`
3. Add the tool in the appropriate category in `README.md`.
4. Update `docs/index.html` with the tool data if applicable.
5. Commit your changes: `git commit -m "Add ToolName to CategoryName"`
6. Push to your fork: `git push origin add-tool-name`
7. Open a pull request.

### Tool Entry Format

Each tool entry should follow this format:

```markdown
- **[Tool Name](https://tool-url.com/)** - Brief description of what the tool does. (Open Source / Commercial / Freemium)
```

### Requirements for New Tools

- **Real tools only** - The tool must exist and be publicly accessible.
- **Working URL** - The link must point to the official website or repository.
- **Accurate description** - Keep it concise (one sentence) and factual.
- **Correct category** - Place the tool in the most relevant category.
- **Pricing model** - Include one of: `Open Source`, `Commercial`, `Freemium`, `Free`, or `Research`.
- **No duplicates** - Check that the tool is not already listed.
- **Alphabetical order is not required** - Tools are ordered by relevance and popularity.

### Quality Standards

- Tools should be actively maintained (updated within the last 12 months).
- Open-source tools should have at least 100 GitHub stars or be from a reputable organization.
- Commercial tools should be publicly available (not invite-only beta).
- Tools must be primarily focused on voice or audio AI.

### What We Do Not Accept

- Tools that are no longer maintained or have been deprecated.
- Tools that are purely conceptual or research papers without implementations.
- Self-promotional submissions without genuine value to the community.
- Duplicate entries or tools that are forks without significant improvements.

## Reporting Issues

- **Broken links** - Open an issue with the `broken-link` label.
- **Incorrect information** - Open an issue or PR with the correction.
- **Tool removed/deprecated** - Open an issue so we can update the list.

## Code of Conduct

Be respectful and constructive. We are building a community resource that benefits everyone in the AI voice and audio space.

## Questions?

Open a [discussion](https://github.com/awesome-ai-tools/curated-ai-voice-tools/issues) if you have questions about contributing.
